Sound_start_recording

The sound_start_recording plug-in provide the possibility to record audio from a microphone or other line-in device. For this item to work correctly, please make sure that the microphone or line-in port is registered as the default sound input device of your operating system. You can change this in the sound settings of your OS. Recordings can be saved either in uncompressed wave (.wav) format or in MPEG Layer-3 (.mp3). Sound will be recorded until a stop_sound_recording item is encountered, or if the soundrecorder is stopped in script by calling exp.soundrecorder.stop(). The following options are available:

  • Record sound [yes or no]: Indicates if sound should be recorded and saved to a file. The option "No" is useful when you ar test running your experiment, during which no sound needs to be recorded yet.

  • Channels [mono or stereo]: Indicates if the recording should consist of one channel (mono) or two channels (stereo). One channel will result in a smaller file size.

  • Sample rate [11025, 22050 or 44100]: The sample rate of the audio file. The higher the number, the better the audio quality.

  • Output Folder/File: The location at which the audio file should be saved. This location is relative to the folder in which the file of the current experiment is located. You may specify several subfolders if you prefer and if these folders do not yet exist, they are created for you. An example entry could be:

    recordings\experiment_X\filename.mp3

    You can use variable names in this field too, delimited by [ ]. For example:

    recordings\subj_[subject_nr]\trial_[count_trial_sequence]

    The extension of the file will automatically determined by the selected option of compression.
  • Compression [wav or mp3]: Indicates whether the audio file should be saved as an uncompressed .wav file (larger file size) or a compressed .mp3 file (smaller file size). If you want to play back recordings by using the OpenSesame sampler item, you should choose wav as the sampler item is not able to playback MP3s. The extension of the filename will automatically be determined by the value that is selected in this option.

  • If file exists [overwrite or append suffix to filename]: Indicates what action to take if a file with the name specified in Output Folder/File already exists.With overwrite selected, the file is simply overwritten and the old file will be lost. If append suffix to filename is selected, a number will be appended to the file when it is saved(e.g. output_1.mp3, output_2.mp3 and so on)

Accessing the soundrecorder in inline_script

When the sound recorder is currently recording, it will be available under the variable self.experiment.soundrecorder (or exp.soundrecorder for OpenSesame > 0.25).

Useful functions are:

soundrecorder.is_recording(): Returns True if it is currently recording and False otherwise.

soundrecorder.stop(): Does the same as the item stop_sound_recording. If this function is called when the soundrecorder is currently recording, it wil stop doing so and write the recorded audio to a file. Please note that it is not possible to resume recording with a soundrecorder item once it has been stopped. You will have to use a new soundrecorder item to make a subsequent recording.
